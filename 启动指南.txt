================================================================================
                    AI短剧自动生成系统 - 启动指南
================================================================================

适用范围: Windows 11 / macOS / Linux
前提条件: Python 3.10+, Docker Desktop, Git Bash (Windows)


================================================================================
一、环境准备
================================================================================

1.1 安装 Python 依赖
--------------------------------------------------------------------------------
  打开终端，进入项目目录：

    cd "C:\Users\86136\Desktop\基于Celery分布式任务队列的AI短剧自动生成系统"

  安装依赖包：

    pip install -r requirements.txt

  验证安装：

    python -c "import celery; print('Celery', celery.__version__)"


1.2 启动 Redis (Docker)
--------------------------------------------------------------------------------
  确保 Docker Desktop 正在运行，然后：

    docker-compose up -d redis

  验证 Redis 是否正常：

    python -c "import redis; r=redis.Redis(host='localhost',port=6379); print(r.ping())"

  期望输出: True


1.3 启动 Flower 监控面板 (可选)
--------------------------------------------------------------------------------
  docker-compose up -d flower

  或者本地启动:

    python -m celery -A celery_app flower --port=5555

  浏览器打开: http://localhost:5555


================================================================================
二、启动 Worker
================================================================================

2.1 启动全部 Worker (推荐入门方式)
--------------------------------------------------------------------------------
  新开一个终端：

    python cli.py worker --type all

  输出示例:

    ==================================================
      启动 Worker: all
      监听队列: ['copywriting', 'image_gen', 'video_gen']
    ==================================================
    -------------- drama-all@HOSTNAME v5.6.3 (recovery)
    ...
    [queues]
      .> copywriting   exchange=copywriting(direct) key=copywriting
      .> image_gen     exchange=image_gen(direct) key=image_gen
      .> video_gen     exchange=video_gen(direct) key=video_gen
    [tasks]
      . tasks.copywriting.generate_script
      . tasks.image_gen.generate_images
      . tasks.video_gen.generate_video
    ...
    drama-all@HOSTNAME ready.


2.2 分别启动专用 Worker (生产推荐)
--------------------------------------------------------------------------------
  终端 A - 文案 Worker:

    python cli.py worker --type copywriting

  终端 B - 图片 Worker:

    python cli.py worker --type image_gen

  终端 C - 视频 Worker:

    python cli.py worker --type video_gen

  每个 Worker 可以独立设置并发数:

    python cli.py worker --type image_gen --concurrency 4

3.4 API 检索推荐
--------------------------------------------------------------------------------
  在接入真实 API 之前，先检索当前市场上最好的模型：

    # 检索全部三类 (文案/图片/视频)
    python cli.py scout

    # 只查某一类
    python cli.py scout --category copywriting
    python cli.py scout --category image_gen
    python cli.py scout --category video_gen

  工具会自动搜索 DuckDuckGo，从搜索结果中提取模型名、版本号、价格，
  输出 Top 2 推荐（首选 + 预算备选），并附带来源链接。

3.4 Web UI (浏览器界面)
--------------------------------------------------------------------------------
  无需 CLI，在浏览器中直接操作：

    python cli.py web

  或者自定义端口：

    python cli.py web --port 9090

  浏览器打开: http://localhost:8080

  Web UI 功能:
    - 左侧卡片：输入主题 → 点击 Generate → 自动走完三阶段流程
    - 流程指示器：实时展示当前进度 (Script Generation → Image Generation → Video)
    - 右侧卡片：任务历史列表，每 3 秒自动轮询刷新状态
    - 结果展示：视频播放器 + 场景图片画廊 + 旁白文本
    - 手动保存：三阶段完成后出现保存面板，输入路径(如 D:\素材\雪山落日)
      → 点击 Save 下载图片到本地、写入 script.txt 和 result.json

  一键启动 (Windows):
    双击项目根目录的 start.bat，自动检查 Redis → 安装依赖 → 启动 Worker → 启动 Web UI

  ⚠ 注意：Web UI 需要 Worker 正在运行才能执行任务。


3.5 查询任务状态
--------------------------------------------------------------------------------
  python cli.py status <task_id>

  示例:

    python cli.py status 2b31ce03-01bb-48a9-bdec-0b648788fdfd

  输出:

    任务ID: 2b31ce03-...
    状态: SUCCESS
    详情: {'title': '命运的重逢', 'scenes': [...], ...}

四、监控与观察
================================================================================

4.1 Flower Web 面板
--------------------------------------------------------------------------------
  浏览器访问: http://localhost:5555

  可查看:
    - Dashboard: 整体任务统计 (成功/失败/重试/活跃)
    - Workers: 在线 Worker 列表及资源使用
    - Tasks: 已注册的所有任务类型及参数
    - Monitor: 实时任务执行流 (成功为绿色，失败为红色)
    - Task 详情页: 点任意任务查看入参、返回值、执行耗时、重试历史


4.2 Redis 命令行监控
--------------------------------------------------------------------------------
  # 进入 Redis
  docker exec -it drama-redis redis-cli

  # 查看队列长度
  LLEN copywriting
  LLEN image_gen
  LLEN video_gen

  # 查看所有 Celery 相关 key
  KEYS celery-*

  # 查看特定任务
  GET celery-task-meta-<task_id>

五、停止系统
================================================================================

  # 停止 Worker: 在 Worker 终端按 Ctrl+C
  # 停止 Docker 服务:
  docker-compose down

  # 停止并清理数据卷:
  docker-compose down -v

6.1 创建新后端 (自定义)
--------------------------------------------------------------------------------
  如需接入其他 API，在 ai_backends/ 目录新建文件，例如 openai.py:

    from .base import BaseAIBackend
    import openai

    class OpenAIBackend(BaseAIBackend):

        def __init__(self):
            self.client = openai.OpenAI(api_key="your-key")

        def generate_script(self, topic: str) -> dict:
            response = self.client.chat.completions.create(
                model="gpt-4o",
                messages=[{"role": "user", "content": f"写一个短剧剧本，主题：{topic}"}],
            )
            # 解析 response 并返回 dict
            ...

        def generate_image(self, prompt: str, index: int) -> dict:
            response = self.client.images.generate(
                model="dall-e-3", prompt=prompt, size="1024x1024"
            )
            return {"scene_id": index+1, "url": response.data[0].url, ...}

        def generate_video(self, scenes, images) -> dict:
            # 调用视频生成 API
            ...

6.2 注册后端
--------------------------------------------------------------------------------
  在 ai_backends/__init__.py 的 _registry 字典中添加:

    from .openai import OpenAIBackend

    _registry = {
        "mock": MockAIBackend,
        "openai": OpenAIBackend,
    }

6.3 使用新后端
--------------------------------------------------------------------------------

================================================================================
七、常见问题
================================================================================

  Q: Worker 启动后提示 "DuplicateNodenameWarning"
  A: 有残留 Worker 进程。关闭所有终端中的 Worker (Ctrl+C)，然后
     ps aux | grep python  查找并 kill 残留进程后重启。

  Q: 任务一直处于 PENDING 不执行
  A: 检查 Worker 是否启动并 listening 对应队列。
     确认 Redis 连接正常: python -c "import redis; r=redis.Redis(); r.ping()"

  Q: 提示 "Connection refused" 连接 Redis 失败
  A: Redis 未启动。执行 docker-compose up -d redis

  Q: Docker Desktop 未运行怎么办
  A: 启动 Docker Desktop 应用程序，等待状态栏图标变为稳定状态。
     如果无法使用 Docker，也可安装 Windows 版 Redis 或使用 WSL。

  Q: 如何修改重试次数和间隔
  A: 编辑 config.py 中的 task_max_retries 和 task_default_retry_delay。
     也可以在单个任务装饰器上自定义 max_retries 和 default_retry_delay。

  Q: Mock 后端返回的数据每次都一样
  A: Mock 后端返回固定示例数据，仅用于功能验证。修改 ai_backends/mock.py
     中的 SAMPLE_SCRIPTS 可以自定义示例内容。或接入真实 AI API (见第六章)。
